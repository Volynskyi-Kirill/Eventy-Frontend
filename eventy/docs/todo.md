# refactor/heavy-calculation-to-backend

**src/lib/api/tickets.service.ts**

**Функция:** groupTicketsByDateAndZone(tickets: Ticket[])

**Что делает:**

- Группирует массив билетов сначала по дате, затем по зоне, а затем формирует итоговую структуру для UI.
- Использует несколько вложенных reduce и map для обработки потенциально больших массивов.

**Почему стоит перенести на бэк-энд:**

- Если количество билетов большое (сотни и тысячи), такие группировки и агрегации могут заметно тормозить фронт.
- Это классический пример бизнес-логики, которую лучше реализовать на сервере (например, через SQL GROUP BY или агрегацию в API).

- Группировка билетов по событию: `const ticketsByEvent = userTickets.reduce...`
- Разделение на будущие и прошедшие события: `Object.entries(ticketsByEvent).forEach...`
- Сортировка событий по дате: `upcomingEvents.sort...`, `pastEvents.sort...`

**Что заменить/перенести на бэк-энд:**

- Группировку билетов по событиям, разделение на будущие/прошедшие и сортировку по дате лучше реализовать на бэке. API должен возвращать структуру `{ upcomingEvents: [...], pastEvents: [...] }`, где каждый элемент уже содержит сгруппированные билеты по событию.
